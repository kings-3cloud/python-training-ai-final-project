### Step 3 — Build the MCP Server tools (one at a time)

### 3a — fetch_url_content tool:

```bash
In `study-assistant/mcp_server/tools/fetch_url.py`, implement the `fetch_url_content` tool.

Requirements:
- Use `httpx` to fetch the text content of a given URL
- If the URL ends in `.pdf`, use `pypdf2` (PyPDF2) to extract text from the downloaded bytes
- Return a plain string of the extracted text (truncated to 4000 chars if longer)
- Handle errors gracefully: return an error string instead of raising

Do NOT wire this into the MCP server yet. Just implement the function and include a small `if __name__ == "__main__"` test block.
```

### 3b — generate_quiz tool:

```bash
In `study-assistant/mcp_server/tools/quiz.py`, implement the `generate_quiz` tool.

Requirements:
- Accept `topic: str` and `num_questions: int` (default 5) as parameters  
- Call the Foundry-hosted model via the OpenAI-compatible API (use `AGENT_ENDPOINT` and `DefaultAzureCredential` from .env, same auth pattern as `computer-history-client/agent_client.py`)
- System prompt: "You are a quiz generator. Return a JSON array of {question, options: [A,B,C,D], answer} objects."
- Parse and return the quiz as a Python list of dicts
- Include error handling and a test block
```

### 3c — save_progress tool:

```bash
In `study-assistant/mcp_server/tools/progress.py`, implement the `save_progress` tool.

Requirements:
- Accept `topic: str`, `score: int`, `total: int` as parameters
- Read/write to `study-assistant/data/progress.json`
- Store entries as: { "topic": ..., "score": ..., "total": ..., "timestamp": ISO8601 string }
- Append new records (don't overwrite old ones)
- Return a summary string like "Saved: 4/5 on Python Basics"
- Include error handling and a test block
```
