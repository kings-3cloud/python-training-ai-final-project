### AI Reponse:
What we're building:

MCP Server — a standalone Python server that exposes three tools over the Model Context Protocol:

fetch_url_content — fetches and returns text content from a URL
generate_quiz — generates quiz questions from provided content
save_progress — persists study progress to a local file
Agent App — a Flask web app using the same AgentClient pattern as computer-history-client/, connected to a Foundry-hosted agent that is configured to call those MCP tools

Folder structure to be created inside ai-final/study-assistant/:

```bash
ai-final/
└── study-assistant/
    ├── mcp-server/
    │   ├── server.py            # MCP server — registers & runs the 3 tools
    │   ├── tools/
    │   │   ├── fetch_url.py     # fetch_url_content tool implementation
    │   │   ├── generate_quiz.py # generate_quiz tool implementation
    │   │   └── save_progress.py # save_progress tool implementation
    │   ├── data/
    │   │   └── progress.json    # persisted study progress (written by save_progress)
    │   └── requirements.txt     # mcp, httpx, beautifulsoup4, etc.
    │
    ├── agent-client/
    │   ├── agent_client.py      # AgentClient class (mirrors computer-history-client pattern)
    │   ├── app.py               # Flask app with chat UI route
    │   ├── .env                 # AGENT_ENDPOINT (not committed)
    │   ├── requirements.txt     # flask, openai, azure-identity, python-dotenv, etc.
    │   ├── static/
    │   │   ├── script.js        # fetch-based chat JS (mirrors existing pattern)
    │   │   └── style.css        # chat UI styles
    │   └── templates/
    │       └── index.html       # Jinja2 chat template
    │
    └── README.md                # setup & run instructions for both parts
```

Key design decisions mirroring the reference:

AgentClient wraps openai.OpenAI with DefaultAzureCredential, same as computer-history-client/agent_client.py
Flask app.py uses the same markdown → bleach sanitization pipeline
.env holds AGENT_ENDPOINT; the MCP server URL is configured on the Foundry agent side, not in the client
The MCP server runs as a separate process (stdio or HTTP transport), decoupled from the Flask app
